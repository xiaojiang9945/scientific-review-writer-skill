# Review Workflow

Use this workflow for full scientific review manuscripts and major revisions.

## Phase Pipeline

1. **Intake**: define topic, discipline, target venue, output language, output format, word count, figure count, citation style, and available source files.
2. **Research**: build search queries, collect or ingest literature, record access limits, and select a core evidence set.
3. **Architecture**: design title options, thesis, outline, word allocation, figure roles, tables, and claim map.
4. **Argumentation**: convert evidence into section-level claims, counterpoints, uncertainty boundaries, and synthesis transitions.
5. **Drafting**: write in English by default, section by section, with citations near supported claims.
6. **Display Items**: generate or revise figures and tables from the written sections and available data.
7. **Citation Audit**: verify citation/reference parity, order, metadata, DOI availability where required, and target style.
8. **Review Loop**: run internal critique, fix critical issues, and document limitations.
9. **Finalization**: deliver Markdown plus DOCX/PDF when requested or feasible.

## Checkpoints

Use checkpoints for high-impact decisions:

- Confirm the thesis and scope before large literature searches.
- Confirm the outline before full drafting.
- Confirm target journal and reference style before final formatting.
- Confirm unresolved limitations rather than hiding them.
- Keep private demos and pre-release repositories private until the owner approves publication.

## Revision Workflow

1. Classify the requested change: scope, evidence, logic, language, citations, figures, tables, formatting, or policy.
2. If a new evidence axis is needed, run a targeted literature refresh.
3. Revise the outline, abstract, figures, and outlook together when the thesis changes.
4. Rerun the citation audit and display-item audit.
5. Record remaining limitations in a QC note, not in the formal manuscript text unless the target genre requires it.
