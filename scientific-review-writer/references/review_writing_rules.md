# Review Writing Rules

## Synthesis Rules

- Build the manuscript around a thesis, not a topic list.
- Separate concept definition, measurement, mechanism, exceptions, applications, and outlook.
- Do not let one convenient assay, data type, organism, period, country, method, or case redefine a broad topic.
- Use exceptions to refine the model rather than placing them as late caveats.
- Make uncertainty testable: name the missing dataset, experiment, benchmark, cohort, perturbation, or validation design.

## Paragraph Rules

- Start with the claim or tension.
- Support with evidence and citation.
- Explain what the evidence means.
- State boundary or implication.
- Transition to the next section when needed.

## Revision Rules

- If a user says the manuscript is shallow, inspect whether the outline lacks a mechanism, taxonomy, exception section, or evidence axis.
- If the user says figures are decorative, rebuild figure roles from the section claims and user data.
- If the user says references are wrong, run citation/reference parity and target-style normalization before further prose editing.
- If the user says the manuscript feels generic, add domain-specific evidence, terminology, and unresolved controversies rather than stronger adjectives.
