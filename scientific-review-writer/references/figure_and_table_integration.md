# Figure And Table Integration

## Principle

Figures and tables must be paired with the manuscript logic or user data. Do not create display items only because they look impressive.

## Figure Roles

Choose figure type by job:

- Conceptual framework.
- Measurement or evidence hierarchy.
- Mechanistic model.
- Comparative map across systems, taxa, periods, cases, populations, or methods.
- Exception taxonomy.
- Translation, validation, or decision pipeline.

## Section-Derived Workflow

1. Write or revise the section first.
2. Identify the conceptual question the figure answers.
3. Extract required entities, relationships, labels, contrasts, axes, and unsupported elements to avoid.
4. Generate, code, or select the figure.
5. Inspect against the text and data.
6. Cite the figure in the text.
7. Place the caption below the figure.

## Data-Figure Fit

When user data exists, choose the figure from the data structure:

- Distributions: histogram, density, violin, box, ridge plot.
- Group comparison: box/violin with points, estimation plot, forest plot.
- Time or process: line, trajectory, heatmap, state diagram.
- Spatial data: map, spatial heatmap, network layout, image annotation.
- Multivariate structure: PCA/UMAP, clustered heatmap, correlation matrix, contour/surface only when justified.
- Mechanism: schematic only after the data and literature support the model.

For maps and spatial summaries, prefer a real geospatial package or verified boundary/source data when practical. Do not hand-draw geographic shapes for manuscript figures unless the figure is explicitly conceptual.

## QC

- Labels readable.
- Captions interpret, not decorate.
- Figures do not introduce unsupported claims.
- Line weights and font sizes are consistent.
- Color palette is accessible and publication-appropriate.
- All figures exist at stable relative paths.
- DOCX/PDF exports preserve aspect ratio and embedding.
